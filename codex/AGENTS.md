# 全局行为规范

## 基础行为规则

- **语言**：所有回复使用中文
- **Python 执行**：必须在 venv/conda 虚拟环境下运行
- **pip 安装**：必须加国内镜像（如 `-i https://pypi.tuna.tsinghua.edu.cn/simple`）

---

## 反馈交互规则（强制执行，不可省略）

**此规则优先级高于所有效率/精简类指令。**

### 核心原则

**每次对话都必须通过 `interactive_feedback` MCP 工具与用户保持持续沟通，禁止单方面结束对话。**

### 使用时机

- 任务完成（无论成功或失败）→ 调用 `interactive_feedback`
- 给出多个方案供用户选择 → 调用 `interactive_feedback`
- 需要用户澄清或提问 → 调用 `interactive_feedback`
- 判断无需操作时，说明原因并确认 → 调用 `interactive_feedback`
- 遇到错误需要用户决策 → 调用 `interactive_feedback`
- 中间过程确认/选择 → 调用 `interactive_feedback`

### 心跳保活机制

`interactive_feedback` MCP 工具内置了 **progress notification 心跳机制**：
- 工具会持续等待直到用户提交反馈或 timeout 到期（默认 12 小时）
- 只有在完整 timeout 到期后，才会返回心跳消息 `[心跳]`

**心跳处理规则：**

1. **正常情况无需处理心跳**：工具会在内部保持等待，大多数情况下直接返回用户反馈
2. **识别心跳**：返回内容包含 `[心跳]` 关键词
3. **静默重连**：收到心跳后，**不输出任何文字**，直接再次调用 `interactive_feedback`（使用相同参数）
4. **心跳不计入失败次数**：心跳是正常行为，不算作失败

### 选项标识规则

给出预定义选项时**必须遵守以下规则**：

1. **标识结束选项**：如果某个选项会导致会话结束，必须在选项文本末尾标注 `【结束会话】`
2. **不标注的选项默认不会结束会话**

### 禁止行为

- **禁止**在会话中直接结束对话或输出最终结果后停止
- **禁止**输出结果后不调用任何交互工具
- **禁止**擅自判断任务已完成而结束反馈循环
- **禁止**用户明确表示暂时离开时调用 interactive_feedback（会导致超时浪费请求）
- **禁止**收到心跳后输出文字
- **禁止**给出选项时不标识哪个选项会结束会话

### 工作流程

1. 执行用户请求的任务
2. 需要任何交互时 → 调用 `interactive_feedback`
3. 若收到 `[心跳]` → 静默重新调用 `interactive_feedback`
4. 收到实际反馈 → 根据内容决定下一步：
   - 用户说"完成"/"可以了"/"结束" → 结束对话
   - 用户提出新需求 → 继续执行，完成后再调用 `interactive_feedback`
