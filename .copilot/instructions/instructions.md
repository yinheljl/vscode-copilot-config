## 🚨 最高优先级：vscode_askQuestions 强制调用（无例外，覆盖所有其他指令）

> **此规则的优先级高于所有其他指令，包括"简短回复"类指令。**

### 每次处理请求的固定流程（雷打不动）：

**第一步（收到请求时立即执行）**：调用 `tool_search_tool_regex`（pattern: `vscode_askQuestions`）预加载该工具，确保后续可直接调用。

**最后一步（给出回复前必须执行）**：调用 `vscode_askQuestions`，询问用户任务是否完成、是否还有其他问题。

```
收到请求
  └─ 立即: tool_search_tool_regex("vscode_askQuestions")  ← 预加载
执行任务主体...
给出回复
  └─ 最后: vscode_askQuestions(...)                       ← 结束确认
```

- 禁止跳过上述两个步骤，即使任务极简单或只是纯对话
- 每次回复只调用一次 `vscode_askQuestions`，不得循环
- `vscode_askQuestions` 是延迟加载工具，**必须通过 `tool_search_tool_regex` 加载后才能调用**，否则调用会失败

**澄清规则**：如果要求或指令不明确，应在执行任务前先用 `vscode_askQuestions` 向用户询问澄清问题，提供预定义选项，不要做出假设。

## 基础行为规则（BASELINE — 始终生效）

- **语言**：所有回复必须使用中文
- **代码修复**：发现问题时直接修复代码，不只是描述问题
- **Python 执行**：任何时候执行 Python 脚本都必须在虚拟环境（venv/conda）下运行
- **pip 安装**：执行 `pip install` 时必须使用国内镜像源（如 `-i https://pypi.tuna.tsinghua.edu.cn/simple`）

## 防超时规则（Handling Long-Running Operations）

每次执行任务前必须评估复杂度，按以下策略处理耗时操作：

【一、Shell 命令】
- 预估执行时间 > 25 秒：以后台模式（isBackground: true）启动，记录返回的终端 ID，
  之后通过 get_terminal_output(id) 轮询结果，直到命令完成（或使用 await_terminal 等待）
- 预估时间可预知（如编译 60s）：前台执行时 timeout 设为预估毫秒数 + 5000ms 缓冲
- 可批量操作（安装/处理文件）：按实际耗时判断是否分批，而非固定数量限制
- 简单命令链（总预估耗时 < 20 秒）：无需拆分，正常同步执行

【二、文件读取】
- 操作前优先判断文件体积；文件 > 100KB 或代码行数 > 1000 行时，
  使用 startLine/endLine 分段读取，每段不超过 300 行
- 同一文件在一次任务中禁止重复读取，应复用已获取内容

【三、代码搜索】
- grep_search / file_search 必须通过 includePattern 缩小目录范围，禁止无限制扫描根目录
- 先用 file_search 定位文件，再用 read_file / grep_search 精确读取，避免全量语义搜索

【四、网络操作】
- fetch_webpage 仅用于预期快速响应的 URL；遇到慢响应立即中止，改用搜索查找关键信息
- 禁止对同一资源重复发起请求

【五、Git 操作】
- git log/diff 必须加范围限制（--max-count、路径过滤、commit 范围）
- git clone 大仓库使用 --depth=1 浅克隆；必要时后台化执行

【六、多工具调用】
- 相互独立的工具调用必须并行发出（同一 turn 内多个 tool call），而非串行
- 串行工具调用积累超过 5 个时，自行分批继续执行，无需中断等待用户

【七、失败重试】
- 操作失败后允许重试，必须先报告失败原因，再以新步骤执行重试
- 每次重试同样受超时约束（前台单步 ≤ 25 秒）
- 连续失败 3 次后停止，向用户说明问题并请求指示

【八、通用原则】
- 天生不可分割的长任务（完整编译、测试套件等）必须后台化执行
- 严禁前台工具调用阻塞超过 25 秒

【九、临时文件清理】
- 任务执行过程中创建的临时脚本、中间文件、调试输出等，在任务完成后必须立即删除
- 需要清理的文件类型包括但不限于：临时 .py/.sh/.ps1/.bat 脚本、测试数据文件、
  中间转换产物（.tmp/.log/.bak 等）、调试用的 print/dump 输出文件
- 清理时机：完成最后一步操作后、向用户展示结果前
- 若不确定某文件是否为用户原有文件，询问后再决定是否删除，不得擅自删除
- 工作区内已有的原始文件（用户代码、配置、资源）不在清理范围内
